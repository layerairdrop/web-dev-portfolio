Multi-Wallet Strategy: Cara Farming Airdrop Tanpa Kena Sybil

Sybil detection adalah musuh utama airdrop farmer. Project semakin pintar mendeteksi wallet yang dikelola satu orang. Artikel ini bahas cara menjalankan multi-wallet dengan aman dan efisien.

Apa Itu Sybil Detection?

Sybil detection adalah teknik yang digunakan project crypto untuk mendeteksi apakah banyak wallet dikelola oleh satu orang. Tujuannya: mencegah satu orang mengklaim reward berkali-kali.

Project menggunakan berbagai sinyal untuk deteksi:

SinyalRisikoSolusi
IP Address sama🔴 TinggiProxy residential
Funding source sama🔴 TinggiMulti-source funding
Timing pattern🟡 SedangRandom delay
Browser fingerprint🟡 SedangAnti-detect browser
Gas price bidding🟢 RendahRandom gas
Transaction pattern🟡 SedangVariasi task order

Layer 1: Network Isolation

Ini layer paling dasar dan paling penting. Setiap wallet harus terlihat seperti orang berbeda dari sisi jaringan.

Proxy Strategy

const proxies = [
  'http://user:pass@residential1:8080',
  'http://user:pass@residential2:8080',
  'http://user:pass@residential3:8080',
];

// Assign proxy per wallet (bukan per session)
const walletProxyMap = {};
wallets.forEach((wallet, i) => {
  walletProxyMap[wallet.address] = proxies[i % proxies.length];
});
⚠️ Jangan pakai proxy datacenter! Project bisa detect IP datacenter dengan mudah. Gunakan residential proxy — lebih mahal tapi jauh lebih aman. Idealnya: 1 proxy per 3-5 wallet.

Browser Fingerprint

Setiap browser punya "sidik jari" unik — user agent, screen resolution, timezone, font, WebGL renderer. Anti-detect browser seperti AdsPower atau Multilogin bisa membuat profile browser unik per wallet.

Layer 2: On-Chain Isolation

Bahkan dengan IP berbeda, project bisa mendeteksi pola on-chain yang mencurigakan.

Funding Source Diversity

Jangan kirim ETH dari satu exchange ke semua wallet. Gunakan strategi "tree":

Exchange
├── Wallet A (funding langsung)
├── Wallet B (funding langsung)
└── Wallet C (funding dari A, bukan exchange)
    ├── Wallet D (funding dari C)
    └── Wallet E (funding dari C)

Dengan struktur ini, tidak ada pola "satu sumber → banyak wallet" yang mudah dideteksi.

Transaction Timing

Jangan lakukan semua task di waktu yang sama untuk semua wallet. Gunakan delay acak:

function getRandomDelay(min, max) {
  return Math.floor(Math.random() * (max - min + 1) + min);
}

// Delay 30-120 detik antar wallet
for (const wallet of wallets) {
  await completeTasks(wallet);
  await sleep(getRandomDelay(30000, 120000));
}

Layer 3: Behavioral Diversity

Layer ini sering dilupakan tapi sangat penting. Orang yang berbeda punya perilaku berbeda.

Variasi Task Order

Jangan semua wallet mengerjakan task dalam urutan yang sama. Acak urutan task:

// JANGAN: semua wallet urut 1→2→3→4→5
// LAKUKAN: setiap wallet urut berbeda
const taskOrder = shuffle([1, 2, 3, 4, 5]);
for (const task of taskOrder) {
  await completeTask(wallet, task);
}

Variasi Gas

Jangan semua wallet menggunakan gas price yang sama persis. Tambahkan sedikit variasi:

const baseGas = await provider.getFeeData();
const variance = Math.random() * 0.1; // 0-10% variance
const customGas = baseGas.gasPrice * (1 + variance);

Variasi Interaction Pattern

  • Beberapa wallet: langsung complete semua task
  • Beberapa wallet: complete 3 task, istirahat 1 jam, lanjut
  • Beberapa wallet: complete task dalam 2 hari
  • Beberapa wallet: skip 1 task (tidak semua task harus 100%)

Wallet Management System

Dengan 50+ wallet, kamu butuh sistem yang terstruktur:

{
  "project": "example-airdrop",
  "wallets": [
    {
      "address": "0x...",
      "proxy": "http://proxy1:8080",
      "funding_source": "exchange",
      "tasks_completed": ["twitter", "telegram", "swap"],
      "tasks_pending": ["bridge", "liquidity"],
      "last_active": "2026-07-06T10:30:00Z",
      "status": "active"
    }
  ]
}
